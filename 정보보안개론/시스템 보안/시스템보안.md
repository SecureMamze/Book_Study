# 시스템의 이해
## 1. 시스템과 네트워크 🍎
**시스템**   
:하나의 연결체를 점, 두개이상의 연결체를 잇는 관계를 선이라 한다면, 점과 선이 연결된 모습을 **시스템**이라고 부른다.   
- 시스템의 형태
    : 과거에는 *하나의 하드웨어가 설치된 하나의 시스템* 이였으나, 네트워크가 강해지고, 시스템간의 경계가 모호해지면서 오늘날의 시스템은 완전한 점의 형태가 아닐 때가 많다.   
    특히 분산처리 시스템이나, 클라우드 환경과 같은 경우에는 더욱 그렇다. 

**시스템 보안**   
: 권한이 없는 사용자가 파일이나 폴더, 장치등을 사용하지 못하게 제한하여 시스템을 보호하는 기능.   

**시스템 보안을 위한 관리🏰**
1. 계정 관리
:   적절한 권한을 가진 사용자를 식별하는 **가장 기본적인 인증수단인  ID, PW** 로, 이를 통한 계정 관리를 시스템의 시작으로 본다.   
>***신분이 확실한 자만이 성에 들어갈 수 있다.***
2. 세션 관리
: *세션(사용자와 시스템 사이 or 두 시스템 사이의 활성화된 접속)관리는 일정한 시간이 지나면 세션을 종료하고 비인가자의 세션 가로채기를 통제한다.
>***외부인의 성 채류 허가시간 관리, 외국 여행시 부여되는 비자.***
3. 접근 제어
: 네트워크 안에서 시스템을 다른 시스템으부터 적절히 보호할 수 있도록 네트워크 관점에서 접근을 통제한다.
>***신분증, 공문서에 따라 접근여부를 결정.***
4. 권한 관리
: 시스템의 각 사용자가 적절한 권한으로 적절하게 정보 자산에 접근할 수 있도록 통제한다.
>***성안에서 각 사람별 행동 권한.***
5. 로그 관리
:시스템 내부나 네트워크를 통해 외부에서 시스템에 어떤 영향을 미칠 경우 그 내용을 기록하여 관리한다.
>***성 안에서 이루어지는 모든 활동을 기록하여 관리***
6. 취약점 관리
:위 다섯가지 관리가 잘 이루어져도 시스템에 보안 문제가 발생할 수 있다. 이는 시스템 자체의 결함에 의한것으로, 이러한 결함을 체계적으로 관리하는것이다.
> ***성벽보수, 신분증 위조여부 조사 , 비리 판별 등***

## 보안 전문가라면 위 6가지의 시선에서 바라보는 관점을 길러야한다.

## 2. 계정 관리🪪
계정은 시스템에 접근하는 가장 기본적인 방법이다.
- 계정의 기본 구성 요소 : ID , PW

### ID 의 속성
: 어떤 시스템에 로그인 하기위해 자신이 누구인지 알리는 과정 -> 식별 과정.   
식별과정 속에서  아이디라는 문자열을 통해 그사람이 누구인지 판별하는 과정은 **정확한 식별과정이 아니다.**

정확한 식별과정   
: 인간의 경우 생체 정보를 기반으로 한것이 정확한 식별 과정에 해당한다.   
ex) 교통계에서 신호위반 차량 운전자의 운전면허증과 운전자의 얼굴을 비교하여 식별.    
하지만, 시스템에서는 생체 정보 확인이 어렵기 떄문에 로그인을 허용하기 위한 확인, 즉 **인증을 위해 PW를 요구한다.**

패스워드는 *보안의네가지인증방법중 *'알고있는 것'* 을 이용한다.

**보안의 네가지 인증방법**
1. 알고있는것 -> 머릿속으로 기억 -> PW
2. 가지고 있는것 -> 직접 소유 -> 신분증, OTP 장치, 출입카드 등
3. 자신의 모습 -> 홍채 등 생체정보 -> 홍체인식, 지문인식
4. 위치하는 곳 -> 현재 접속을 시도하는 위치의 적절성 확인 -> *콜백(접속을 요청한 사람의 신원을 확인하고, 미리 등록된 전화번호로 전화를 되걸어 접속을 요청한 사람이 본인인지 확인하는 과정)

## 2-1. 운영체제의 계정 관리
운영체제   
: 시스템을 구성하고 운영하기 위한 기본적인 수단. 운영체제에 대한 권한을 가지게 되면 해당 시스템에서 동작하는 다른 응용 프로그램에 대해서도 어느정도의 권한을 가질 수 있다.   
ex) 운영체제를 탈취하면 운영체제속에서 작동되는 프로그램 (웹서버, 데이터베이스, 응용프로그램 등) 을 조작하거나 정지시킬 수 있다. 
-> 다시말해 상당한 수준의 권한을 가지고 있다는 뜻.   
운영체제 관리자로부터 응용프로그램에 있는 기밀정보, 인증 정보 등을 보호하는 방법은, **응용프로그램을 암호화 하는것이다.**

*운영 체제의 계정을 관리할때, 관리자 계정 뿐만아니라, 대부분의 문서를 읽을 수 있는 일반 계정또한 주시해야한다.

1. 윈도우의 계정 관리   
    - 운영체제 관리자 권한 계정 :  administrator (윈도우 설치시 가장 기본으로 설치되는 계정)   
```net localgroup administrators``` 명령 사용시 관리자 그룹에 속하는 각계정의 존재형태 확인 가능   
 *사용자 계정을 모두 확인하려면 ```net users``` 명령을 사용한다.   

    운영체제 계정관리시 그룹또한 고려해야한다.
    윈도우에서 시스템 존재그룹 목록을 확인하려면, ```net localgroup``` 명령어를 실행한다.

    1-1. 윈도우의 주요 그룹
    - Administrators(대표적 관리자 그룹, 모든 권한 소유)
    - Power Users(로컬 컴퓨터에서만으로 제한되는 관리자 그룹)
    - Backup Operators(윈도우 시스템에서 시스템 파일을 백업하는 권한, 로컬에 로그인하고 시스템을 종료할 수 있음.)   
    - Users(일반유저. 관리자 계정에 비해 한정된 권한, 서버로의 로컬 로그인 가능)   
    - Guests(Users 그룹과 같은권한. 서버로의 로컬 로그인은 금지됨.) 

2. 유닉스 계정 관리
    - 유닉스 계열 시스템에서는 기본 관리자 계정으로 root 계정이 존재한다.
    유닉스에서는 /etc/passwd 파일에서 계정 목록을 확인할 수 있다.

    **/etc/passwd 파일 구성**   
    ```root : x : 0 : 0 : root : /root : /bin/bash```   
    1. root: 사용자 계정
    2. x: pw가 암호화 되어 shadow 파일에 저장되어있음.
    3. 0: 사용자 번호
    4. 0: 그룹 번호
    5. root: 실제 이름 , 시스템 설정에 영향 X
    6. /root: 사용자의 홈 디렉토리 설정. 위의 예시는 root 계정으로 홈 디렉토리가 /root (/) 로 설정되어있음.
    7. /bin/bash: 사용자의 셀정의 기본 설정은 bash

    유닉스에서 관리자 권한은 3번과 4번으로 식별한다. 관리자는 번호가 0이고, 그룹번호도 0이다.   
    만약 root 이외에 사용자 번호가 0인 계정이 있으면 그 계정 또한 관리자 권한을 가진다.

    유닉스에서는 일반 사용자 계정도 /etc/passwd 파일에서 확인한다. 이때, /etc/passwd 파일 구조에서 2와 7을 확인해야 한다.   
    과거 리눅스나 유닉스에서는 2가 공백이면, 패스워드 없이 로그인이 가능한 계정으로 처리되었으나, 계정의 보안문제로 최근 시스템에서는 **2가 비어있을 경우 해당계정을 비활성화** 한다.
    7또한 /bin/false 처럼 **명시적으로 사용이 금지되어 있거나 비어있다면, 누군가에세 할당된 계정이 아닌 계정이다.**

    유닉스에서는 그룹을 /etc/group 파일에서 확인한다.

    **/etc/group 파일 구성**  
    ```root : x : 0 : root```
    1. root: 그룹이름
    2. x: 그룹 패스워드(일반적으로 사용하지 않음)
    3. 0: 그룹 번호 -> 0은 root 그룹
    4. 해당 그룹에 속한 계정 목록. 하지만 이목록은 완전하지 않으므로 패스워드 파일과 비교해보는 것이 가장 정확.

    유닉스에서도 root를 제외한 그룹은 윈도우 처럼 **모두 임의로 생성되며 고유 권한이 없음.**

확인된 계정을 관리하는 방법.   
- 보안상 문제가 되는 계정은 '계정을 가지고 있을 필요가 없는 사람이 가진경우' 뿐이다.    
그러므로 계정 생성과 삭제시의 적절한 승인절차가 마련되어야하고, 불필요한 계정이 존재하는지, 불필요한 관리자 권한등이 부여되는지를 주기적으로 확인해야 한다.

## 2-2. 데이터 베이스의 계정 관리
: 데이터 베이스 또한 운영체제와같이 계정이 존재.   
데이터베이스 < 운영체제 의 형식으로 이루어져 있어 운영체제 계정을 관리 못하면, 데이터베이스의 계정 관리또한 소용 없다.
